Requisitos inversos

Requisitos funcionais

  • O sistema deve gerenciar o acervo
  • O sistema deve vericar quais livros disponíveis
  • O sistema deve conter um espaço para reserva de exemplares
  • O sistema deve conter lembrete de devolução
  • O sistema deve permitir que o usuário registre sugestões de livros
  • O sistema deve conter um relatório de materiais mais utilizados
  • O sistema deve monitorar o uso da biblioteca
  • O sistema deve conter um ranking dos livros mais lidos
  • O sistema deve ter um espaço para discussão sobre os livros
  • O sistema deve registrar quais livros cada aluno está lendo e por quanto tempo

Requisitos não funcionais

  • O acesso deve ser restrito a professores e alunos
  • Os alunos precisam ter liberdade para acessar o sistema em qualquer lugar e horário
  • O sistema pode ter um espaço para os professores indicarem leituras personalizadas
  • O sistema pode permitir que os professores anexem atividades complementares aos livros recomendados
  • Os alunos podem reservar livros com antecedência
  • A biblioteca tem que ser bem planejada e estruturada
  • O sistema pode ter um ranking dos livros mais lidos
  • O diretor pode ter um painel de monitoramento para a biblioteca que funcione de forma eficiente

Requisitos inversos

  • O sistema não pode permitir que as obras fiquem retidas por muito tempo
  • O sistema não pode deixar que o usuário esqueça a data de devolução
  • O sistema não pode deixar que pessoas externas utilizem os recursos indevidamente
  • O sistema não pode ser de difícil acesso
  • O sistema não pode disponibilizar livros já reservados
